基于 Gist(GitHub) + Settings Sync 实现 VSCode 工作站配置同步

发表于 2017-12-18 | 更新于: 2019-09-19 | 分类于 IDE | 阅读热度: 次
字数统计: 863 | 阅读时长 ≈ 3

写在前面：

VSCode 上有各种各样不同的插件，如果要在不同的电脑上分别重复进行插件的安装以及配置是件比较麻烦的事情。Settings Sync 可以将使用的插件以及其配置保存到 GitHub 上，以后只需要从 GitHub 上获取，就可以一次性安装插件配置信息，而且配置过程简单，这无疑是振奋人心的。

适用于

电脑更换时还要一个个去搜索插件安装，想要实现多个设备 VSCode 环境配置同步

你得提前准备：

  • 注册申请 GitHub 账号
  • 安装了 Git
  • 安装好 VSCode

1 >> 可同步什么？

Setting Sync 可同步包含的所有扩展和完整的用户文件夹：

  • 设置文件
  • 快捷键设置文件
  • Launch File
  • Snippets Folder
  • VSCode 扩展设置
  • 工作空间
  • ……

2 >> Setting Sync 快捷键

Sync: Update / Upload Settings >>> Shift + Alt + U（上传）

Sync: Download Settings >>> Shift + Alt + D（下载）

注意：如果快捷键有冲突，可使用 Ctrl + K + S 快捷键搜索查询冲突快捷设置。

2 >> Github Login

如果没有进行 Settings Sync 配置的话，VSCode 重启后会自动打开 Settings Sync 欢迎页面（或使用快捷键 Shift + Alt + U），然后点击【LOGIN WITH GITHUB】使用 GitHub 进行登录:

点登录后自动弹出 GitHub 登陆界面，登录并授权成功后如下:

3 >> Gist ID && Token

上诉 Login 成功之后，会弹出一个 Select Your Existing Gist 界面：

上述操作之后会在 Github 中生成一个 Gist 私有库（带有标识 Gist Id，类似于：4d22267358XXXXXXXXXXX），然后关闭窗口。如果没有生成，你可以点击【SKIP（NEW ONE WILL BE CREATED UPON FIRST UPLOAD）】重新生成一个。

此时事实上备份配置已经完成，你可以从 Settings Sync 欢迎页面点击【EDIT CONFIGURATION】查看完整配置（关键参数：Gist ID 以及 Acccess Token）：

注意，你的 Gist ID 即为 Gist 私有库标识，你可以打开 Github Gist 库重新获取到（URL）。
