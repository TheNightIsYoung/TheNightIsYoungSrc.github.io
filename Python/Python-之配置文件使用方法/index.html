项目中常常会使用到大量和配置相关的常量，我们将这些配置项抽取到在一个文件中，这就是配置文件，它将项目代码中所有的配置项都变成模块化配置的。本文我们来看 Python 中配置文件的使用方法。

Python 中配置文件策略

我们知道，配置文件会将项目代码中所有的配置项都变成模块化配置的。那么，让配置项 “模块化可配置的” 有什么好处？

1. 为什么要使用配置文件？

配置文件在项目中是非常必要的。

当我们需要修改配置项时，不需要涉及到代码内部的修改，避免了项目中代码文件对常量的分散使用，让配置常量可以统一修改，避免造成修改不全面的问题，大大提高了代码的重用性，方便后期软件项目的维护。

2. Python 中的配置文件使用方法

我们先来盘点一下 Python 中主流的使用配置文件的方法：

2.1 –> ConfigParser

Python3 自身提供了标准的配置读写模块 ConfigParser。导入方式如下：

Python中十分常见的配置文件格式有：.ini、.cfg、.conf、.py，当然，还有的使用 txt、.json 等。

并且，配置文件一般会以 config 进行命名，便于理解和使用。

事实上，.ini、.cfg、.conf 以及 txt 格式的配置文件使用方法都是一致的，只存在一个后缀的区别，但本质上都是 text 文本文件。

而 .py 配置文件，严格来说不是配置文件，是一个扩展包，在 Python 项目中使用时作为一个本地包导入即可。

故，后文 ConfigParser 相关小节，我们主要就 ConfigParser 模块操作 .ini、.cfg、.conf 以及 txt 等文本格式的配置文件，实现配置文件的读取、更新、写入、删除等进行解读。

2.2 –> YAML

有些人可能还见过，yaml 或者 yml 后缀的文件，什么鬼？事实上，它是一种由 YAML 语言写成的配置文件。

YAML 是专门用来写配置文件的语言，非常简洁和强大，远比 JSON 格式方便。
