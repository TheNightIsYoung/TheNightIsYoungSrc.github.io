继 TensorFlow1.0 之后，Google 于 2016 年 推出了一个名为 slim 的库。TensorFlow-Slim 是基于 TensorFlow 的一个高级 API 接口（轻量级库），它将许多常见的 TensorFlow 函数进行了二次封装，使得代码变得更加简洁，特别适用于构建复杂结构的深度神经网络。

TensorFlow-Slim

TF-Slim 是 TensorFlow 中一个用来构建、训练、评估复杂模型的轻量化库（高级 API 接口）。TF-Slim 模块可以和 TensorFlow 中其它 API（例如：tf.contrib.learn）混合使用。

官方文档地址：https://github.com/tensorflow/tensorflow/tree/master/tensorflow/contrib/slim

3. Slim 模块的优点

上面我们已经比较了 TensorFlow 原始 API 和 TF-Slim 库实现卷积的难易程度。确实，使用 TF-Slim 可以帮助我们使得模型的构建、训练、评估变得简单：

  1. 允许用户用极其简洁的代码定义模型，这主要由 arg_scope、高级的 layers 和 variables 工具模块来实现。这些工具增加了代码的可读性和维护性，减少了复制、粘贴超参数值出错的可能性，并且简化了超参数的调整。
  2. 通过提供常用的 regularizers 来简化模型的开发。
  3. 正如前面提过的，很多常用的计算机视觉模型（例如 VGG、AlexNet）在 Slim 里面已经有了实现。这些模型开箱可用，并且能够以多种方式进行扩展（例如，给内部的不同层添加 multiple heads）。
  4. TF-Slim 库的使用，使得 “复杂模型的扩展” 及 “从一些现存的模型的 checkpoints 开始训练” 变得容易。

4. Slim 模块组件

TF-Slim 是独由几个独立的模块（模块化的）组成：

  1. arg_scope 模块：允许用户对该 scope 内的操作定义默认参数；
  2. data 模块：包含了 Slim 模块的 dataset definition、data providers、parallel_reader 以及 decoding utilities；
  3. evaluation 模块：评估模型相关；
  4. layers 模块：构建模型所需要的一些高级 layers；
  5. learning 模块：训练模型相关；
  6. losses 模块：常见的 loss 函数定义相关；
  7. metrics 模块：常见的评估指标相关。
  8. nets 模块：所实现的经典深度卷积网络（例如 VGG、AlexNet）。注意：最新的 Slim 中已经没有 nets 模块了！！！
  9. queues 模块：提供一个简易的用于开始和关闭 QueueRunners 的上下文管理器（content manager）；
  10. regularizers 模块：权重正则化器（regularizer ）相关；
  11. variables 模块：封装简化了变量创建和操作。

注意：在原生的 TensorFlow 中，可以分为两种类型的变量：regular variables（常规变量） 和 local (transient) variables（临时变量）。绝大部分的变量是 regular variables，它们一旦被创建，他们能够用一个 Saver 保存到 disk（可以持久化）；而 local variables 只存在于一个 session（运行期）中，不会被持久化到文件。

–>不同的是

TF-Slim 通过定义 model variables 进一步区分了变量，模型变量（model variables）代表一个模型的参数。这些模型变量在学习阶段被训练或微调，在评估和预测阶段从 checkpoint 中加载（即可训练变量），例如，由 slim.fully_connected 和 slim.conv2d 函数所创建的变量。

而非模型变量（Non-model variables）是指在学习或评估阶段使用，但不会在预测阶段起作用的变量，例如，global_step 变量在训练评估中需要，推理时不需要。类似地，moving average variables（滑动平均变量）可以 mirror（映射）模型变量，但是它们本身不是模型变量。

8. 使用 Slim 评估模型（Evaluating Models）

事实上，有了 5~7 小节中的内容，你已经可以进行深度学习快速开发实验。

一旦我们训练好了一个模型（或者模型还在训练中），我们想看一下模型在实际中性能如何。TF-Slim 库中可以通过获取一系列表征模型性能的评估指标（evaluation metrics）来实现。

评估代码一般会加载数据，执行前向传播，和 Ground Truth（真实标记）进行比较并记录评估分数。这个步骤可能执行一次，也可能周期性地执行。

8.1 Slim 评价指标（Metrics）

比如我们定义了一个不是损失函数的性能度量指标（损失在训练过程中进行直接优化），但是这个评价指标对于我们模型的评估十分重要。

比如说，我们想最小化对数损失（log loss），但是我们感兴趣的指标可能是 F1 score（test accuracy），或者 IoU 分数（Intersection Over Union score，这个指标不可微，因此不能作为损失）。

TF-Slim 中提供了一系列评价指标操作（metric operation），这使得模型评估更简单。理论上，计算一个评价指标的值可以分为三步骤：

  1. Initialization：初始化用于计算指标的变量；
  2. Aggregation：执行用于计算指标评价所需的操作（比如：sum 等）；
  3. Finalization：收尾（可选），执行其他用于计算指标值的操作。例如，取 mean、min、max 等。
