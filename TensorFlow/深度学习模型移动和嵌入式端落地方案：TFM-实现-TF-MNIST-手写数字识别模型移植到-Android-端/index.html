TFM 实现 TF MNIST 手写数字识别模型移植到 Android 端

本文我们来看 TFM MNIST Demo：利用 TensorFlow 实现一个 MNIST 手写数字识别模型并进行持久化，然后把持久化的模型移植到 Android 端（SAMSUNG Galaxy S7）上运行。

正如官网给出的关于 “Android TensorFlow support” 描述的：

这里，我们首先给出 TFM 移植方案的一般思路：

一般先利用 Bazel 工具把 TensoFlow 编译成 .so 库文件 和 jar 包，进行 Android 相关配置，即可实现 TF 模型的移植。

Bazel 编译成功后，”.so library” 和 “Java JAR” 分别对应的具体文件名称如下：

  1. libtensorflow_inference.so
  2. libandroid_tensorflow_inference_java.jar

将 1、2 导入 Android 平台后便可以使用其中提供的接口调用 TF Model。

–> 注意：

手动使用 Bazel 编译 TensoFlow 过程中可能遇到很多问题，导致最终编译 .so 库文件 和 jar 包 失败，这是很常见的！！！事实上，不会使用 Bazel 编译也没关系，TensoFlow 官方已经为开发者提供了编译好的 .so库文件 和对应的 jar 包 文件，下载后即可使用。

这里先不要急，后面我会给出上述的两种 “.so library” 和 “Java JAR” 获取方式的具体说明。

2. Moblile Phone 调用 MNIST Model 完成移植过程

上面我们给出 TFM 移植方案的一般思路，这里我们来看移动或嵌入式端掉用 TF 模型完成移植的过程简介：

  1. 获取训练好的 TF Model，并且明确模型网络节点信息（尤其是输入输出节点），完成模型预测可用验证；
  2. 在 Android Studio 中创建 Android 项目，导入 Android 平台调用 TF 模型需要的 jar 包和 so 文件 (负责 TF 模型的解析和运算)，以及 TF Model ，并完成相关的项目配置；
  3. 创建并完成调用：定义变量用于储存模型输入输出，并通过 jar 包提供的接口进行 TF 模型的调用。还需要简单注意与 Android UI 页面的关联。
  4. 移植测试

下面，我们将根据上述移植过程简介来完成 Moblile Phone 调用 TF Model 完成移植的全过程：
